# Plan B-2.5 · Layout Chrome Migration — Design Spec

## Goal

Migrate the i-SMARTLUPUS app chrome (Sidebar, BottomNav, protected layout outer shell) off hardcoded blue-and-white styling onto the emerald + dark-mode token system established in Plan A/B-1/B-2. Add a togglable (expand/collapse) desktop sidebar, a sticky header with page title + theme toggle, and a mobile hamburger drawer for secondary actions.

## Why

Plan B-2 surfaced two real gaps the existing plan sequence didn't cover:

1. **Dark mode is broken on app chrome.** Bento content, pages, and primitives all tokenize correctly. But `components/Sidebar.tsx`, `components/BottomNav.tsx`, and `app/(protected)/layout.tsx` still use `bg-white`, `text-blue-600`, `bg-slate-50` — so toggling dark mode leaves a light sidebar and nav wrapping dark content. Looks broken.
2. **No sidebar collapse, no sticky header, no mobile drawer.** The hospital-staff user isn't going to want a fixed 256px sidebar eating screen real estate on a tablet workflow. And there's no sticky title bar, so scrolling a long page loses all orientation.

This plan closes those gaps before continuing the page-by-page migration (Plans B-3 through B-8).

## Non-goals (explicit)

- Migrating `components/ui/bottom-nav.tsx` primitive (dead code from Plan A that never got wired up) — reconcile in a separate post-B-8 cleanup
- Page-level header customizations (e.g. filter toolbar on `/semua`) — sticky header stays basic
- Gesture-based drawer dismiss (swipe-left) — tap-backdrop only for v1
- Sidebar drag-to-resize
- Breadcrumbs in the header — we explicitly chose minimal header (title only)
- Migrating any of the 6 remaining pages (mohon / semakan / semua / status / pengguna / profil) — those are Plans B-3 through B-8

## Architecture

The protected layout becomes a 3-piece composition:

```
ProtectedLayout
├── Sidebar              (desktop only, md+)
│   ├── Header: brand + chevron-toggle
│   ├── Nav: role-aware items
│   └── Footer: user avatar + name
├── Main
│   ├── AppHeader        (sticky, both breakpoints)
│   │   ├── Desktop: chevron-toggle + page title + ThemeToggle
│   │   └── Mobile: hamburger + page title + ThemeToggle
│   └── {children}       (page content)
├── BottomNav            (mobile only, <md)
└── MobileDrawer         (rendered but closed; opened by hamburger)
    ├── Brand + role chip
    ├── User avatar + name
    ├── ThemeToggle (duplicate for discoverability)
    └── Log Keluar button
```

**State management:**
- Sidebar collapse state: `useState` in layout, synced to `localStorage("sidebar-collapsed")`
- Drawer open state: `useState` in layout
- Route changes auto-close the drawer (via `usePathname` effect)
- Theme state: unchanged — already managed by `ThemeProvider`

**Interaction summary:**
- Desktop user clicks chevron in sidebar header → sidebar transitions to 64px icon rail, labels hide. Clicks chevron again → expands back to 240px. State persists across reloads.
- Mobile user taps hamburger in sticky header → drawer slides in from left with backdrop. Taps backdrop OR taps any nav item in BottomNav (which causes route change) → drawer closes.
- Scrolling long pages → sticky header stays pinned, title + theme toggle always reachable.

### `components/AppHeader.tsx`

```ts
interface AppHeaderProps {
  pageTitle: string;
  sidebarCollapsed: boolean;    // drives chevron direction on desktop
  onToggleSidebar: () => void;  // wired to desktop chevron button
  onOpenDrawer: () => void;     // wired to mobile hamburger button
}
```

Two separate callbacks (not one `onPrimaryAction`) keep intent explicit and let the layout hook each breakpoint's trigger to the right state setter without branching on viewport.

Renders (one container, two triggers — only one visible per breakpoint via Tailwind class):
- Desktop button (`hidden md:inline-flex`): `<button onClick={onToggleSidebar}>` with `<ChevronLeft />` (if expanded) or `<ChevronRight />` (if collapsed)
- Mobile button (`inline-flex md:hidden`): `<button onClick={onOpenDrawer}>` with `<Menu />` (hamburger)
- Title: `<h1 className="text-subhead font-semibold">{pageTitle}</h1>`
- Right: `<ThemeToggle />`

Styling: `sticky top-0 bg-[var(--surface)]/85 backdrop-blur border-b border-[var(--border)] px-4 h-14 flex items-center gap-3 z-30`.

### `app/(protected)/layout.tsx` (rewrite)

```tsx
"use client";

import { useEffect, useState } from "react";
import { usePathname, useRouter } from "next/navigation";
import { createClient } from "@/lib/supabase/client";
import type { Profile } from "@/lib/supabase/types";
import Sidebar from "@/components/Sidebar";
import BottomNav from "@/components/BottomNav";
import { AppHeader } from "@/components/AppHeader";
import { MobileDrawer } from "@/components/MobileDrawer";
import { getPageTitle } from "@/lib/page-title";

const COLLAPSE_KEY = "sidebar-collapsed";

export default function ProtectedLayout({ children }: { children: React.ReactNode }) {
  const router = useRouter();
  const pathname = usePathname();
  const [profile, setProfile] = useState<Profile | null>(null);
  const [loading, setLoading] = useState(true);
  const [collapsed, setCollapsed] = useState(false);
  const [drawerOpen, setDrawerOpen] = useState(false);

  // Restore collapse state
  useEffect(() => {
    const stored = localStorage.getItem(COLLAPSE_KEY);
    if (stored === "true") setCollapsed(true);
  }, []);

  // Auto-close drawer on route change
  useEffect(() => {
    setDrawerOpen(false);
  }, [pathname]);

  // Profile load (unchanged logic)
  useEffect(() => {
    async function loadProfile() {
      const supabase = createClient();
      const { data: { user } } = await supabase.auth.getUser();
      if (!user) { router.push("/login"); return; }
      const { data } = await supabase.from("profiles").select("*").eq("id", user.id).single();
      if (!data) { router.push("/login"); return; }
      setProfile(data as Profile);
      setLoading(false);
    }
    loadProfile();
  }, [router]);

  function toggleCollapsed() {
    const next = !collapsed;
    setCollapsed(next);
    localStorage.setItem(COLLAPSE_KEY, String(next));
  }

  async function handleLogOut() {
    const supabase = createClient();
    await supabase.auth.signOut();
    router.push("/login");
  }

  if (loading) {
    return (
      <div className="min-h-screen bg-[var(--bg)] flex items-center justify-center">
        <p className="text-sm text-[var(--fg-muted)]">Memuatkan...</p>
      </div>
    );
  }
  if (!profile) return null;

  const pageTitle = getPageTitle(pathname);

  return (
    <div className="min-h-screen bg-[var(--bg)] text-[var(--fg)]">
      <Sidebar
        role={profile.role}
        name={profile.full_name}
        collapsed={collapsed}
        onToggleCollapsed={toggleCollapsed}
      />
      <main className={collapsed ? "md:ml-16" : "md:ml-60"}>
        <AppHeader
          pageTitle={pageTitle}
          sidebarCollapsed={collapsed}
          onToggleSidebar={toggleCollapsed}
          onOpenDrawer={() => setDrawerOpen(true)}
        />
        <div className="pb-20 md:pb-8 px-4 py-6 max-w-5xl mx-auto">
          {children}
        </div>
      </main>
      <BottomNav role={profile.role} />
      <MobileDrawer
        open={drawerOpen}
        onOpenChange={setDrawerOpen}
        role={profile.role}
        name={profile.full_name}
        onLogOut={handleLogOut}
      />
    </div>
  );
}
```

Note: prop API for AppHeader is specified above (two callbacks + title + collapse flag). No further API decisions deferred.

## Dark-mode token audit

Before this plan: chrome is broken in dark mode. After this plan, the following MUST use tokens (no hardcoded colors):

| Surface | Before | After |
|---------|--------|-------|
| Sidebar bg | `bg-white` | `bg-[var(--surface)]` |
| Sidebar border | `border-r` (slate default) | `border-[var(--border)]` |
| Sidebar brand text | `text-blue-600` | `text-[var(--primary)]` |
| Sidebar nav active | `bg-blue-50 text-blue-700` | `bg-[var(--primary-tint)] text-[var(--primary)]` |
| Sidebar nav inactive | `text-slate-600 hover:bg-slate-50` | `text-[var(--fg-muted)] hover:bg-[var(--primary-tint)] hover:text-[var(--fg)]` |

| BottomNav border | `border-t` | `border-t border-[var(--border)]` |
| BottomNav active | `text-blue-600 bg-blue-50/50` | `text-[var(--primary)] bg-[var(--primary-tint)]` |
